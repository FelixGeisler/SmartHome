= ADR 1: Serve the SPA from the Spring Boot jar

== Status
Accepted

== Context
The dashboard is an SPA and the only client of the REST API. A separate origin or reverse proxy would add CORS and independently deploys we don't need.

== Decision
The Maven build compiles the SPA into the jar under classpath:/static. Spring serves it at /, with non-/api routes falling back to index.html. In development, the Vite dev server proxies /api to Spring.

== Consequences
One artifact, no CORS, UI and API are always on one version. Cost: a frontend-only change rebuilds the whole jar, and the build fetches Node on a cold cache (skip with -Dskip.frontend=true).
