= 5. Building Block View

[plantuml,building-blocks,svg]
----
@startuml
skinparam componentStyle rectangle

[frontend] #line.dashed
[web]
[device]
package "integration" {
  interface "DeviceAdapter\n(port)" as port
  [registry]
  [adapter]
}
database "Database" as db
node "Device" as dev

[frontend] --> [device] : REST (/api)
[web] ..> [device] : maps errors
[device] --> db : JPA
[device] --> [registry] : route command
[registry] --> port
[adapter] ..|> port
[adapter] --> dev
@enduml
----

[cols="1,3",options="header"]
|===
| Block | Responsibility
| `frontend` | The device dashboard: a React SPA (own toolchain under `hub/frontend/`) that registers, lists, and toggles devices through the REST API. It only sees the REST contract, never the domain model.
| `web` | Maps exceptions to RFC 9457 `ProblemDetail` responses.
| `device` | Application core: the REST API plus register / list / toggle logic and persistence.
| `integration` | The `DeviceAdapter` port and the registry that routes to it; one adapter per device technology plugs in here.
|===
